# Rakugo Schedule Web

個人利用向けの Web ページ構想です。日本の落語公演を、公式サイト中心の公開情報から集約して一覧化します。

今回は要件変更に合わせて、iPhone アプリではなく「サーバーなし」「手元の大規模な開発環境なし」でも作りやすい静的 Web ページ前提で整理しています。

## 目的

- 寄席、ホール落語、独演会を 1 つのページで見られる
- 日付、会場、出演者、エリアで探せる
- 元情報へすぐ飛べる
- 個人利用として無理なく維持できる

## この条件での現実的な結論

最初に向いているのは、以下の構成です。

- フロントエンドは `HTML + CSS + JavaScript` のみ
- 配信は静的ホスティング
- データは `events.json` のような静的 JSON を読む
- ブラウザから直接各公式サイトをスクレイピングしない

この形なら、独自サーバーや複雑な開発環境がなくても進めやすいです。

## なぜこの構成か

ブラウザだけで他サイトを直接読みにいく方法は、`CORS`、HTML 構造差、更新失敗時の切り分けの問題が大きく、安定しません。個人利用でも、まずは「収集済みデータを読む静的ページ」に分けるのが安全です。

## MVP 要件

### 必須

1. 落語公演の一覧表示
2. 日付順ソート
3. 日付、カテゴリ、会場、出演者の絞り込み
4. 公演詳細の表示
5. 情報ソースへの外部リンク
6. 最終更新日時の表示

### あったほうがよい

1. 今日、今週末などのクイックフィルタ
2. お気に入り
3. ブラウザの `localStorage` を使った保存

### 後回しでよい

1. 会員機能
2. チケット購入の一元管理
3. 地図連携の高度化
4. 自動スクレイピングの完全自動化

## 画面要件

### 1. 一覧画面

- 今日以降の公演を日付順で表示
- カードかテーブルで見やすく表示
- 各行に日付、タイトル、会場、出演者、カテゴリを表示

### 2. 絞り込み UI

- 日付範囲
- カテゴリ
- 会場
- 出演者名検索
- エリア

### 3. 詳細表示

- 公演名
- 開催日時
- 会場
- 出演者
- 料金
- 元情報 URL
- チケット URL
- 最終取得日時

## 技術要件

### 採用

- 素の `HTML`
- 素の `CSS`
- 素の `JavaScript`
- データファイルは `JSON`

### 採用しない

- Node.js 前提のビルド
- 専用バックエンド
- データベース必須の構成
- React / Next.js のような重い開発基盤

## データ更新要件

この条件では、更新方法を 2 段階で考えるのが現実的です。

### 方式 A: 手動更新

- 公式サイトを見て JSON を手で更新する
- 最初の試作に最も向く
- サーバー不要

### 方式 B: 無料サービスで半自動更新

- GitHub Actions
- Google Apps Script
- Cloudflare Workers

ただし、MVP ではまず手動更新で十分です。

## 配信候補

- GitHub Pages
- Netlify
- Cloudflare Pages

いずれも静的サイトなら独自サーバー不要で始められます。

## テスト要件

ローカルの重い開発環境がなくても、以下で十分に進められます。

1. `index.html` をブラウザで開いて見た目を確認
2. 複数条件で絞り込みが正しく動くか手で確認
3. JSON の読み込み失敗時の表示を確認
4. スマホ幅と PC 幅の両方を確認
